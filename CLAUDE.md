# 🚨【重要: Fail-First原則の厳守】🚨

本プロジェクトでは**Fail-First**（失敗を早期に顕在化させる）原則を厳格に採用します。

- **Fail-Over（失敗の隠蔽・握りつぶし）や安易なリカバリは絶対に禁止**。
- **try/exceptでエラーを握りつぶす実装は禁止**。
- **未知の入力（Unknown inputs）や異常値には、早期に失敗として検知・停止すること**。
- エラーや異常は必ず明示的にraiseし、原因を隠さずに表面化させてください。

> **例外処理は本質的な回復が可能な場合のみ許可し、原則として“失敗に早く気付く”設計を徹底してください。**

# Coding Principles

- **DRY (Don't Repeat Yourself)**  
  重複を避け、既存のコードやライブラリを最大限に活用する。
- **YAGNI (You Aren’t Gonna Need It)**  
  将来必要になるかもしれない機能を先回りして実装しない。今必要なものだけ作る。
- **KISS (Keep It Simple, Stupid)**  
  複雑さを避け、シンプルで明快な実装を優先する。
- **Boy Scout Rule**  
  コードを触ったら来た時よりきれいにする。不要なコードやコメントを削除し、整理する。
- **Dead Code Removal / Code Hygiene**  
  呼ばれないコードや不要になった実装は積極的に削除し、技術的負債を溜め込まない。
- **Technical Debt Awareness**  
  将来の保守コストを意識し、短期的な便宜より長期的な健全性を優先する。
